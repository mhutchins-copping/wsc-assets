### How it works (current state — PR #7 merged + deployed)

1. Admin opens the wizard, picks a person.
2. Picks **iPhone** or **Android**.
3. Picks **Council-owned** or **Staff-owned (BYOD)**.
4. (iPhone + Council) types the serial. Wizard does live ABM lookup
   via Microsoft Graph DEP endpoints:
   - **In ABM** → green status → on Provision, calls
     `assignUserToDepDevice` to pre-bind the user. Setup Assistant
     pre-fills the council username on factory-reset boot.
   - **Not in ABM** → amber status with inline Apple Configurator
     instructions (the iOS app, signed in with mdm@walgett.nsw.gov.au,
     adds consumer-bought iPhones to ABM in ~90 sec each). Plus a
     "skip ABM, use Company Portal" fallback option.
5. Provision creates an asset register entry (`enrolDeviceImpl`) and
   mints a 14-day handover URL with the right OS-specific walkthrough.
   Handover URL is public (token-gated, no SSO) so staff can open it
   on the device itself.
